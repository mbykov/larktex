## 🎯 Цель

Преобразование устного математического текста на русском в LaTeX.

**Пример:**
```
"предел икс стремится к бесконечности от синус икс"
→ "\lim_{x \to \infty} \sin(x)"
```

## 📋 Этап 1: Нормализатор (`normalizer.py`)

**Только простые замены** (без анализа структуры):

| Задача | Пример |
|--------|--------|
| Заменить термины на латиницу | `синус` → `sin`, `икс` → `x` |
| Числа → цифры | `ноль` → `0`, `два` → `2` |
| Степени | `в квадрате` → `^2`, `куб` → `^3` |
| Подразумеваемое умножение | `sin x y` → `sin x * y` |
| `all` в конце → `* 1` | `a + b all` → `a + b all * 1` |

**Не делает**:
- Не добавляет `nall` (это делает парсер)
- Не группировка скобок (это делает парсер)
- Не проверяет ошибки (это делает парсер)

### i18n/ru.json — структура

```json
{
  "variables": {
    "x": ["икс", "икса", "иксу", "иксом", "иксе"],
    "α": ["альфа", "альфы", "альфе", "альфой"]
  },
  "functions": {
    "sin": ["синус", "синуса", "синусу", "синусом"],
    "exp": ["экспонента", "экспоненты", "экспоненте", "exp"]
  },
  "special": {
    "inf": ["бесконечность", "бесконечности", "бесконечностью"],
    "lim": ["предел", "предела", "предель"]
  },
  "relations": {
    "tends_to": ["стремится к", "стремится", "→"]
  }
}
```

**Падежи:** Все падежи указаны явно в массиве синонимов. Нормализатор просто заменяет любой синоним на целевое значение (латинская буква/функция).

## 📐 Этап 2: Парсер (`grammar.lark` + `parser.py`)

**Обработка структуры**:

| Правило | Описание |
|---------|----------|
| **all** | "здесь закрывающая скобка, ищи открывающую влево" |
| **nall** | Автоматически добавляется перед оператором, если нет `all` и есть цепочка (2+ члена) |
| **all в конце** | `* 1` (добавлено нормализатором) |
| **Подразумеваемое `*`** | `sin x y` → `sin x * y` (добавлено нормализатором) |
| **Ошибки** | `all` внутри `sin(...)` или в начале строки → ошибка |

### Алгоритм парсера

1. **Вход:** `sin a + b all * c + d all`
2. **Удалить `all * 1`:** `sin a + b all * c + d`
3. **Добавить `nall`:** `sin a nall + b all * c nall + d`
4. **Обработать `all`:** `sin(a + b) * (c + d)`
5. **Удалить `nall`:** `sin(a + b) * (c + d)`
6. **Вывод LaTeX:** `\sin(a + b) * (c + d)`

### Грамматика Lark

- Функции: `sin`, `cos`, `tan`, `exp`, `sqrt`, `log`, `ln`, ...
- Пределы: `lim [as] x tends_to value of (expr)`
- Степени: `^`, `all` имеет приоритет над `^`
- `all` = явные скобки (наивысший приоритет)

## 🧪 Примеры

| Вход | Нормализация | Результат LaTeX |
|------|--------------|-----------------|
| `синус икс` | `sin x` | `\sin(x)` |
| `синус икс плюс игрек` | `sin x nall + y` | `\sin(x) + y` |
| `синус икс плюс игрек всё` | `sin x + y all * 1` | `\sin(x + y)` |
| `предел икс стремится к бесконечности от (синус икс)` | `lim x tends_to inf (sin x)` | `\lim_{x \to \infty} \sin(x)` |
| `экспонента от икс` | `exp x` | `\exp(x)` |
| `синус квадрат икс` | `sin^2 x` | `\sin^2(x)` |

## 🚀 Как расширить

### Добавить новую функцию
1. Добавить в `i18n/ru.json` → `"exp": ["экспонента", ...]`
2. Добавить в `grammar.lark` → `FUNC_NAME: "exp"`
3. Добавить в `parser.py` → обработка в `_handle_tree()`
4. Добавить в `design.py` → `LATEX_FUNCS.append('exp')`

### Добавить новый оператор
1. Добавить в `i18n/ru.json` → `"tends_to": ["стремится к", ...]`
2. Добавить в `grammar.lark` → `TENDS_TO: "tends_to"`
3. Добавить в `parser.py` → обработка в `comparison()`
