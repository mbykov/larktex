## 🎯 Цель проекта

Преобразование устного описания математических формул на русском языке в LaTeX-код.

**Пример:**
```
Вход:  "синус во второй из переменной икс, всё в квадрате"
Выход: "\sin^2(x)"
```

---

## 🏗 Архитектура

Проект состоит из двух этапов:

```
┌─────────────────────┐     ┌─────────────────────┐     ┌─────────────┐
│   Нормализатор      │ ──▶ │     Lark Parser     │ ──▶ │   LaTeX     │
│  (tokenizer.py)     │     │   (grammar.lark)    │     │   Output    │
└─────────────────────┘     └─────────────────────┘     └─────────────┘
      │                            │
      ▼                            ▼
   i18n/ru.json                AST tree
```

### Этап 1: Нормализатор (`normalizer.py`)

**Задачи:**
1. **Мультиязычность** — словарь синонимов в `i18n/ru.json`
2. **Замена на латиницу** — `синус` → `sin`, `икс` → `x`
3. **Числа в цифры** — `ноль` → `0`, `два` → `2`
4. **Математические острова** — выделение формул из обычного текста

**Примеры нормализации:**

| Вход | Выход |
|------|-------|
| `синус во второй из переменной икс` | `sin^2(x)` |
| `икс равно игрек, из этого следует, что зет равно нулю` | `x = y. z = 0` |
| `А умножить на Б всё плюс В, всё в квадрате` | `A * B all + V all^2` |
| `корень квадратный из а плюс б делить на два` | `sqrt(a + b) / 2` |

### Этап 2: Lark Parser (`grammar.lark` + `parser.py`)

**Задачи:**
1. Построение AST по грамматике
2. Обработка приоритетов операций
3. Обработка конструкции "всё" (маркер области действия)
4. Преобразование AST в LaTeX

**Семантика "всё":**

| Вход | Результат |
|------|-----------|
| `А плюс Б всё делить на В` | `(A + B) / V` |
| `А плюс Б делить на В` | `A + B / V` |
| `А умножить на Б всё плюс В, всё в квадрате` | `((A * B) + V)^2` |
| `А умножить на Б плюс В в квадрате` | `A * (B + V^2)` |

## 🧠 Нормализатор — логика работы

### 1. Выделение математических островов

```python
# Вход: "икс равно игрек, из этого следует, что зет равно нулю"
# Результат:
#   [("икс равно игрек", True),
#    (", из этого следует, что ", False),
#    ("зет равно нулю", True)]
```

**Эвристика:** Сегмент считается математическим, если содержит:
- Известные функции (`синус`, `корень`, `логарифм`)
- Известные переменные (`икс`, `дельта`, `лямбда`)
- Математические операторы (`плюс`, `умножить`, `в степени`)

### 2. Нормализация каждого острова

Последовательность замен:
1. **Числа** — слова → цифры (`ноль` → `0`)
2. **Функции** — синонимы → латиница (`синус` → `sin`)
3. **Переменные** — варианты → латиница (`икс` → `x`)
4. **Операторы** — синонимы → символы (`плюс` → `+`)
5. **Степени** — `квадрат` → `^2`, `куб` → `^3`
6. **"всё"** — маркер области → `all`

### 3. Сортировка по длине ключей

При замене слов важно сначала заменять **длинные слова**, чтобы избежать частичных замен:

```python
# Правильно: ["во второй степени"] → "^2"
# Неправильно: ["степени"] → "^" (оставит "во второй")
sorted_keys = sorted(keys, key=len, reverse=True)
```

## 📐 Грамматика Lark — основные правила

```lark
start: expr

expr: term ("плюс" term | "минус" term)*
term: factor ("умножить на" factor | "делить на" factor)*
factor: primary ("в степени" primary)?

primary: NUMBER
      | VARIABLE
      | "(" expr ")"
      | FUNCTION "(" expr ")"
      | "all" "(" expr ")"  // <-- конструкция "всё"

FUNCTION: "sin" | "cos" | "tan" | "sqrt" | "log" | ...
VARIABLE: /[a-z]+/
NUMBER: /\d+(\.\d+)?/ | "pi" | "e"

%import common.WS
%ignore WS
```

**Приоритеты:**
1. Функции и скобки (высший)
2. Степень
3. Умножение / деление
4. Сложение / вычитание (низший)

## 🧪 Тестовые примеры

### Простые
```
"синус от икс" → "sin(x)"
"косинус квадрат икс" → "cos^2(x)"
"икс в квадрате" → "x^2"
"корень из а плюс б" → "sqrt(a + b)"
```

### С "всё"
```
"А плюс Б всё делить на В" → "(A + B) / V"
"А умножить на Б всё плюс В, всё в квадрате" → "((A * B) + V)^2"
```

### Математические острова
```
"икс равно нулю, из этого следует, что игрек равен единице"
→ "x = 0. y = 1"

"если синус икс равен одному, то икс равен пи делить на два"
→ "if sin(x) = 1 then x = pi / 2"
```

### Логика
```
"для любого эн существует эм такое, что эн больше эм"
→ "for_all n exists m such_that n > m"
```

## 📝 Заметки для продолжения

1. **Текущий статус:** Создана папка `i18n/`, написан словарь `ru.json`.
2. **Следующий шаг:** Реализовать `normalizer.py` согласно логике выше.
3. **Особенности:**
   - "всё" должно остаться как маркер `all` для Lark
   - Обычный текст между островами — оставить как есть
   - Приоритет длинных слов при замене

4. **Проблемы, которые могут возникнуть:**
   - Двусмысленность: "делить на" может быть частью дроби или оператора
   - Контекст: "зет" может быть переменной или буквой в тексте
   - Падежи: "синус от икса" vs "синус икс"

5. **Решения:**
   - Использовать слово "от" как маркер аргумента функции
   - Контекст: если после слова следует оператор — это переменная
   - Нормализовать падежи: `икса` → `икс`, `игреку` → `игрек`
