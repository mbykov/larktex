# Правила синтаксиса LarkTex

Это документ описывает правила, по которым работает LarkTex. Обратите внимание: **правила представлены неформальным языком для понимания логики**, а не как строгая спецификация. Реальная реализация находится в `lib/grammar.lark` и `lib/normalizer.py`.

---

## 1. Правило `all` (группировка выражений)

Ключевое слово `all` (или `всё` в русском тексте) означает **"здесь должна быть закрывающая скобка"**.

### Как это работает:

1. **`all` действует влево** — находит последовательность переменных или функций перед собой
2. **Открывающая скобка** ставится в начале этой последовательности
3. **Если `all` в конце выражения** → добавляется виртуальное `* 1`

### Примеры:

| Вход | Объяснение | Результат |
|------|------------|-----------|
| `a + b all` | Группируем `a + b` | `(a + b)` |
| `a + b all / c` | `(a + b) / c` | `(a + b) / c` |
| `a + b all * c` | `(a + b) * c` | `(a + b) * c` |
| `a + b all * c all ^ 2` | `((a + b) * c)^2` | `((a + b) * c)^2` |
| `x all` | Одна переменная → нечего группировать | `x` |
| `a + b all` в конце | Виртуальный `* 1` | `(a + b) * 1` → `(a + b)` |

### Особые случаи:

- **Если перед оператором нет `all`** → добавляется `nall` (no-all), что означает "не группировать"
- **Если последовательность из одного члена** → ничего не добавляется, нечего группировать
- **`all` в конце выражения или математического острова** → `all * 1` (виртуальное умножение на единицу)

## 1.1 Правило границ для степеней и корней

**Границы выражения в степени или под корнем определяются так:**

1. **`all` / `всё`** — явная граница (закрывающая скобка)
2. **Закрывающая скобка `)`** — естественная граница
3. **Конец выражения** — если нет `all` и скобок, граница — конец
4. **Предлоги `от` / `из`** — для корней указывают начало подкоренного выражения

### Примеры для степеней:

| Вход | Объяснение | Результат |
|------|------------|-----------|
| `a в квадрате` | `a^2` (одна переменная) | `a^2` |
| `a плюс б в степени эн` | Без `all` — только `б` в степени | `a + b^n` |
| `а плюс б всё в степени эн` | `(a+b)^n` | `(a+b)^n` |
| `a в степени три минус пять` | Без `all` — только `3` в степени | `a^3 - 5` |
| `a в степени три минус пять всё плюс два` | `(a^(3-5)) + 2` | `(a^(3-5)) + 2` |

### Примеры для корней:

| Вход | Объяснение | Результат |
|------|------------|-----------|
| `корень из а` | `\sqrt{a}` | `\sqrt{a}` |
| `корень из а плюс б` | Без `all` — только `a` под корнем | `\sqrt{a} + b` |
| `корень из а плюс б всё` | `\sqrt{a+b}` | `\sqrt{a+b}` |
| `корень степени 5 из а плюс б всё` | `\sqrt[5]{a+b}` | `\sqrt[5]{a+b}` |
| `корень степени 5 плюс 7 от три плюс два всё` | `\sqrt[5+7]{3+2}` | `\sqrt[5+7]{3+2}` |

### Ключевое отличие:
- **Без `all`** — степень/корень применяются только к **непосредственно предшествующему** элементу
- **С `all`** — степень/корень применяются к **группированному выражению**

## 1.2 Правило виртуального `* 1` для `all` в конце выражения

**Если `all` стоит в конце выражения или математического острова, это подразумевает виртуальное умножение на 1.**

Это нужно для корректной группировки предшествующей последовательности.

### Примеры:

| Вход | Объяснение | Результат |
|------|------------|-----------|
| `a + b all` | `(a + b) * 1` | `(a + b)` |
| `a^2 * b / c all` | `a^{2 * b / c}` (конец выражения → авто-скобка) | `a^{2 * b / c}` |
| `a^2 * b / c all * x + y all` | `a^{2 * b / c} * (x + y)` | `a^{2 * b / c} * (x + y)` |
| `x all` | `x * 1` | `x` |

**Важно:** Нормализатор не добавляет `* 1` явно, но парсер должен учитывать это правило при обработке `all` в конце выражения.

## 2. Правило виртуального умножения

**Если две переменные или функции разделены пробелом, то между ними добавляется виртуальный знак `*`**.

### Примеры:

| Вход | Объяснение | Результат |
|------|------------|-----------|
| `a b` | `a` и `b` разделены пробелом | `a * b` |
| `sin x cos y` | `sin(x) * cos(y)` | `sin(x) * cos(y)` |
| `2 x` | `2 * x` | `2x` |

## 3. Правила нормализатора

Нормализатор (`lib/normalizer.py`) преобразует русский текст в латинские символы:

### Функции:
- `синус` → `sin`
- `косинус` → `cos`
- `тангенс` → `tan`
- `котангенс` → `cot`
- `арксинус` → `arcsin`
- `арккосинус` → `arccos`
- `логарифм` → `log`
- `натуральный логарифм` → `ln`
- `экспонента` → `exp`
- `корень` / `квадратный корень` → `sqrt`

### Операторы:
- `плюс` → `+`
- `минус` → `-`
- `умножить на` / `на` (в контексте) → `*`
- `делить на` → `/`
- `в квадрате` → `^2`
- `в степени` → `^`
- `равно` → `=`
- `не равно` → `!=`
- `меньше` → `<`
- `больше` → `>`
- `меньше или равно` → `<=`
- `больше или равно` → `>=`

### Особые символы:
- `бесконечность` → `inf`
- `дельта` → `\delta`
- `альфа` → `\alpha`
- `бета` → `\beta`
- `гамма` → `\gamma`
- и другие греческие буквы

### Конструкции:
- `дробь ... на ...` → `frac ..., ...`
  - **Важно:** В конструкции `дробь X на Y`, `X` — числитель, `Y` — знаменатель.
  - Если есть `всё` (`all`), оно указывает границу группировки: `дробь X всё на Y` означает `(X) / Y`.
  - Если в числителе или знаменателе есть `плюс`/`минус`, они оборачиваются в скобки: `дробь 1 на 1 минус p` → `frac 1,(1-p)`.
  - **Требование:** В дроби обязательно должен быть разделитель `на` (или `делить на`, `по`). Без разделителя конструкция считается ошибкой.
  - Примеры: `дробь 1 на 2`, `дробь a минус b на n`, `дробь 1 на 1 минус p`.
- `факториал` → `!`
- `двойной факториал` → `!!`
- `ц из ... по ...` → `binom` (биномиальный коэффициент)
- `а из ... по ...` → размещения
- `интеграл от ... де ...` → `integral ... d ...`
- `предел ... стремится к ...` → `lim ... tends_to ...`
- `производная от ... по ...` → `deriv ...`

---

## 4. Приоритеты операций

1. **Функции и скобки** — наивысший приоритет
2. **Степени** (`^`)
3. **Умножение и деление** (`*`, `/`)
4. **Сложение и вычитание** (`+`, `-`)

Пример: `a + b * c` → `a + (b * c)`

---

## 5. Примеры полных выражений

| Русский текст | Нормализовано | LaTeX |
|---------------|---------------|-------|
| `синус икс` | `sin x` | `\sin(x)` |
| `а плюс б всё делить на в` | `a + b all / v` | `(a + b) / v` |
| `корень из а плюс б` | `sqrt a + b` | `\sqrt{a} + b` |
| `икс в квадрате` | `x^2` | `x^2` |
| `интеграл от икс де икс` | `integral x d x` | `\int x \, d x` |
| `дробь один на два` | `frac 1,2` | `\frac{1}{2}` |
| `эн факториал` | `n!` | `n!` |
| `предел икс стремится к нулю от синус икс` | `lim x tends_to 0 sin x` | `\lim_{x \to 0} \sin(x)` |

## Примечание

Эти правила описаны **неформальным языком** для понимания логики работы системы. Реальная реализация может иметь нюансы, которые определены в:
- `lib/grammar.lark` — формальная грамматика Lark
- `lib/normalizer.py` — словарь замен
- `lib/parser.py` — построение AST
- `lib/generator.py` — генерация LaTeX

Для точной спецификации обращайтесь к исходному коду.
